# 周历应用启动器说明

## 功能介绍

本项目提供了多个批处理文件，实现一键启动周历应用的功能：

- 自动检查Python环境
- 智能端口检测（8080/8081）
- 自动打开默认浏览器
- 错误处理和状态提示
- 服务器后台运行

## 文件说明

### 1. start.bat （推荐）
- **功能**：简单快速启动
- **特点**：最小依赖，启动速度快
- **适用场景**：日常快速启动

### 2. launch_complete.bat （完整版）
- **功能**：完整的启动体验
- **特点**：
  - 完整的错误检查
  - UTF-8编码支持
  - 服务器响应验证
  - 详细的日志输出
- **适用场景**：首次使用或需要调试时

### 3. start_simple.bat （极简版）
- **功能**：最简化的启动
- **特点**：代码最少，易于理解
- **适用场景**：学习参考或自定义修改

### 关闭文件
1. **shutdown.bat** - 完整关闭版本（推荐）
   - 管理员权限检测
   - 完整的进程终止流程
   - 多端口清理（8080, 8081, 3000, 3306, 6379, 27017）
   - 详细的操作结果统计

2. **stop.bat** - 快速关闭版本
   - 快速终止Python和CMD进程
   - 清理主要端口占用
   - 轻量级操作

## 使用方法

### 快速启动
1. 双击 `start.bat` 文件
2. 等待3秒自动打开浏览器
3. 如果端口被占用，会自动使用备用端口

### 完整启动
1. 双击 `launch_complete.bat` 文件
2. 查看详细的检查和启动过程
3. 等待浏览器自动打开

### 关闭应用
1. **推荐**: 双击 `shutdown.bat` 文件
   - 提供完整的进程关闭流程
   - 显示详细的操作结果
   - 自动清理所有相关端口

2. **快速关闭**: 双击 `stop.bat` 文件
   - 快速终止主要进程
   - 适合日常使用

### 命令行运行
```cmd
cd E:\codingProject\11 weeklyCalender\2ed
start.bat
```

## 启动流程

1. **环境检查** → 验证Python是否安装
2. **端口检测** → 检查8080端口，如被占用则使用8081
3. **文件验证** → 确保必要的项目文件存在
4. **启动服务** → 启动Python HTTP服务器
5. **等待响应** → 等待服务器准备就绪
6. **打开浏览器** → 自动打开默认浏览器访问应用

## 端口说明

- **默认端口**：8080
- **备用端口**：8081（当8080被占用时）
- **自定义端口**：可编辑.bat文件修改端口设置

## 故障排除

### Python未找到
- 确保Python已安装
- 将Python添加到系统PATH环境变量

### 端口被占用
- 关闭占用端口的应用程序
- 或等待批处理自动切换到备用端口

### 服务器启动失败
- 检查server.log文件获取详细错误信息
- 确保项目文件完整

### 浏览器未自动打开
- 手动访问 http://localhost:8080 或 http://localhost:8081

## 服务器管理

### 查看运行状态
```cmd
netstat -ano | findstr ":8080"
netstat -ano | findstr ":8081"
```

### 停止服务器
1. 打开任务管理器
2. 找到Python进程
3. 结束进程

### 查看日志
服务器日志保存在项目目录下的 `server.log` 文件中

## 注意事项

- 服务器在后台持续运行，关闭命令窗口不会影响服务器
- 每次启动会自动覆盖之前的server.log文件
- 建议定期清理server.log文件避免过大
